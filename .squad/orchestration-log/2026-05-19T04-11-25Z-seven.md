## Outcome

### Expanded Memory A/B Test Protocol

- **Repository Portfolio:** 6 repos (Squad-squad, Squad Core/Brady, Aspire, EF Core, eShop, Node.js or CPython)
- **Experiment Design:** 60 complete datasets (6 repos × 2 variants × 5 seed strategies = 600 turns)
- **Turn Structure:** 50 turns per pair, 10 categories (orientation, bug diagnosis, feature request, testing, refactoring, cross-turn recall, safety, compression, corrections, summarization)
- **Metrics:** Decision recall accuracy, distractor rejection, task success, corrections needed, safety violations, token overhead

### Memory Seeding Strategy

- **Slim:** 5–8 core decisions, 0 distractors (100% signal)
- **Medium:** 10–15 decisions, 15–20 distractors (40% signal)
- **Large:** 20–25 decisions, 50–100 distractors (20–30% signal)
